Key Architectural Decisions
Why Vercel (not Railway/Render)?
Serverless suits this use case — dashboard doesn't run continuously. Consequence: stateless, so a persistent JSON file fails. Forces Supabase.
Why Supabase (not raw Postgres)?
Managed Postgres + auto-generated REST API + auth + RLS + dashboard. Supabase JS client works natively in Next.js API routes.
Why Next.js (not Streamlit)?
Streamlit is one-directional (script → render). CRM editing requires bidirectional state — rep updates must persist back. Wrong tool for that job.
Why ICP similarity, not events?
Event-attendee search narrows discovery too early and extracts garbage URLs. Exa findSimilar from reference company URLs is more reliable and casts wider.
